# 토스 생태계 한 눈에 이해하기 (오늘의 짝꿍 프로젝트 기준)

"오늘의 짝꿍" 서비스를 성공적으로 런칭하기 위해 사용하는 여러 사이트들이 각각 어떤 역할을 하는지, **'건물 짓기'**에 비유하여 정리해 드립니다.

| 플랫폼 명칭 | 비유 | 주요 역할 (우리가 하는 일) | 현재 단계에서의 의미 |
| :--- | :--- | :--- | :--- |
| **토스 비즈니스 (Toss Business)** | **사업자 면허** | 사업자 인증, 정산 계좌 관리, 파트너 정보 등록 | "나 여기서 장사하는 공식 사업자요"라고 신고하는 곳 (방금 **사업자 등록증** 올리신 곳!) |
| **토스 개발자 센터 (Developers)** | **건축 법규 & 설계도** | API 가이드 확인, 서비스 등록, SDK 다운로드 | "토스라는 도시에서 건물을 어떻게 지어야 하는가?" 공부하고 구경하는 도서관 |
| **토스 콘솔 (Console)** | **공사 현장 관리소** | 빌드 업로드, 버전 관리, 인앱 상품(운세 리포트 등) 등록 | 실제 코드(빌드)를 올리고, 물건(인앱 결제)을 진열대에 올리는 **실무 대시보드** |
| **토스 앱 (Toss App)** | **완성된 매장** | 사용자가 서비스를 이용하고 결제함 | 유저들이 "오늘의 짝꿍"을 만나서 운세를 보는 **최종 무대** |

---

### 💡 나는 어디서 무엇을 하나요? (Workflow)

1.  **토스 비즈니스**에서: "오늘의 짝꿍" 운영 주체(사업자)가 누구인지 인증받습니다. (심사 대기 예정)
2.  **토스 개발자 센터**에서: 토스 디자인 시스템(TDS)이나 결제 API를 어떻게 쓰는지 기술적인 법규를 확인합니다.
3.  **토스 콘솔**에서: 제가 짠 코드를 **빌드**해서 올리고, 유료 운세 리포트 같은 **상품**을 만듭니다.
4.  **토스 앱**에서: 유저가 들어와서 실제로 운세를 보고 기분 좋게 결제합니다.

> [!TIP]
> 지금 사업자 등록 파일을 올리셨으니, 이제 토스로부터 **"당신은 믿을 수 있는 사업자입니다"**라는 승인을 기다리는 단계입니다. 승인이 나면 **토스 콘솔**에서 결제 기능 등을 본격적으로 활성화할 수 있게 됩니다!
