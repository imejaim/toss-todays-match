**날짜:** 2026-01-07
**상태:** 광고 SDK 연동 완료 / 빌드 성공

## 1. 진행된 작업
사용자님의 요청에 따라 **보상형 광고(Rewarded Ad)** 기능을 `PremiumReport` 페이지에 연동했습니다.
이제 사용자가 "무료로 보고서 열기"를 클릭하면 실제 앱인토스 광고가 로드되고, 시청 완료 시 프리미엄 결과가 해금됩니다.

### 변경 사항
- **`src/hooks/useRewardedAd.ts` 생성**: `@apps-in-toss/web-bridge`의 `GoogleAdMob`을 사용하는 커스텀 훅 구현.
- **`src/pages/PremiumReport.tsx` 수정**: 모의(Mock) 광고 코드를 제거하고 실제 광고 훅(`useRewardedAd`) 연동.
- **타입 안정성 확보**: `npm run lint` 통과 (타입 에러 없음).

## 2. 출시 전 필수 확인 사항 (필독)

### ⚠️ 광고 ID 설정 (중요)
현재 코드는 **테스트용 광고 ID** (`ait-ad-test-rewarded-id`)가 설정되어 있습니다.
앱 심사 및 실제 배포를 위해서는 **토스 개발자 콘솔**에서 발급받은 **실제 광고 ID(Placement ID)**로 변경해야 합니다.

1. **토스 개발자 콘솔** > 수익화 > 인앱광고 메뉴 접속.
2. 새 광고 단위 만들기 (보상형 광고).
3. 발급받은 ID를 복사.
4. `src/hooks/useRewardedAd.ts` 파일의 `TEST_AD_GROUP_ID` 값을 변경.
   ```typescript
   const TEST_AD_GROUP_ID = '여기에_실제_광고_ID_입력';
   ```

### ⚠️ 개인정보 처리방침
- `src/pages/Privacy.tsx` 파일 내용을 귀사의 실제 정책에 맞게 수정해주세요 (현재는 템플릿 상태).
- 콘솔의 **앱 정보** > **기본 정보** 탭에서 [개인정보 처리방침 URL]에 `https://{내_도메인}/privacy`를 입력해야 합니다.

## 3. 배포 파일
- **파일명**: `todays-match.ait`
- **위치**: 프로젝트 루트 디렉토리
- **사용법**: 토스 개발자 콘솔 > 버전 관리 > **새 버전 만들기** > `.ait` 파일 업로드.

## 4. 출시 심사 팁
- 심사 제출 시 "테스트 광고 ID" 상태로 제출하면 반려될 수 있습니다. 반드시 **실제 ID**로 교체 후 빌드하여 제출하세요.
- 광고가 로드되지 않는 경우를 대비해 예외 처리가 코드에 포함되어 있으나, 네트워크 환경에 따라 광고 로딩 시간이 걸릴 수 있음을 심사 메모에 남기면 좋습니다.
